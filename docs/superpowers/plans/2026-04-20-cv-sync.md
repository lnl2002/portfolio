# Portfolio ↔ CV Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Align the ported portfolio's text content with `docs/cv.md` (extracted from `public/cv.pdf`), fix factual drift from the design-handoff copy, and expand under-specified sections (RecruitMe stack, FPT outcome) without touching visual design.

**Architecture:** All user-facing copy already lives in `messages/en.json` + `messages/vi.json` (next-intl). Tech-name arrays (stacks, chips) live as constants at the top of each section component. Therefore every content fix is a narrow edit to one or two files, verifiable by an e2e text assertion. No component structure changes; no CSS changes.

**Tech Stack:** Next.js 16 (App Router) · next-intl v4 · Playwright (webpack-backed dev webServer) · bun.
